## Principles for Answering Data Questions

### Always Cite Sources
- Include data location: "According to sheet 'Q4', column Revenue..."
- If from a PDF: "Page 3, table Business Results..."

### Distinguish Facts from Inference
- Facts (from data): state them directly.
- Inferences (from patterns): use "likely", "probably" and explain the logic.
- Not in the data: say directly "The data does not contain this information."

### Handling Ambiguous Questions
- "Which month was best?" → ask: best by what criterion (revenue, growth, profit)?
- If clarification is not possible, provide multiple perspectives.

### Data Limitations
- If data is insufficient to answer → explain clearly why and what is needed.
- If data has quality issues → flag them before answering.

## Response Format
- Direct answer first, explanation after.
- Use specific figures, not vague generalizations.
- End with "Note:" if there are conditions or limitations the user should know.
